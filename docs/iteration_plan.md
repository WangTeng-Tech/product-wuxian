# wuxian-web 项目进度与迭代方案

基于对 `wuxian-web` 项目核心代码（`src/views`, `src/components` 等）和 `README.md` 的代码审查，以下是当前的进度评估及后续迭代建议：

## 一、当前进度评估

1. **整体进度 (生产就绪)**
   - 首页、产品、解决方案等核心页面（如 `Product.vue`, `Products.vue`）均已通过 Vue3 + Element Plus 建设完毕，整体页面完整度极高。

2. **下载中心 (Phase 1 状态)**
   - **现状**：在 `AppDownloadModal.vue` 中已实施严格的“登录墙”。未登录用户仅能看到登录/注册表单，这完全符合阶段一的“严防死守”去黑名单策略。
   - **待办**：`README.md` 中建议的“在登录界面加一行纯文本说明：‘支持 Windows, macOS, Android, iOS 全平台客户端’”尚未在代码中实装。

3. **即时客服系统 (Customer Service)**
   - **现状**：已完全实现。前端通过 `ChatBubble.vue` 和 `ChatWindow.vue` 接入，后端依靠 Supabase Realtime 实现。
   - **管理端**：`Admin.vue` 集成了“在线客服”管理面板，防刷机制也已到位。

---

## 二、后续迭代方案

### 阶段 1：短期优化（完善 Phase 1）
**目标**：提升现有功能细节，不破坏当前的安全审查。

1. **优化下载中心文案**
   - **动作**：修改 `AppDownloadModal.vue`，在 `<p class="auth-subtitle">` 中补充纯文本说明：“支持 Windows, macOS, Android, iOS 全平台客户端”。
   - **目的**：保障用户知情权，提高转化。

2. **修复客服系统未读状态**
   - **动作**：在 `App.vue` 中接入真实的未读消息计算逻辑，替换当前硬编码的 `unreadCount: 0`。

### 阶段 2：中期迭代（过渡至 Phase 2）
**目标**：在确认解除安全警报后，恢复常态化运营的极佳体验。

1. **下载中心交互改版**
   - **动作**：调整 `AppDownloadModal.vue`（或新增组件），默认对外展示各平台的下载按钮，但渲染为 `<button @click="showLogin">`。
   - **逻辑**：只有获取到合法的登录状态后，才将按钮替换为带有真实分发链接的 `<a>` 标签。避免直接暴露 `href="file.exe"`。

2. **管理后台客服功能增强**
   - **动作**：为 `Admin.vue` 的在线客服模块增加“浏览器消息通知 (Notification API)”或提示音功能。
   - **目的**：使管理员能在后台挂机时及时响应访客。

### 阶段 3：业务扩展（代理商伙伴计划分润平台）
**目标**：构建自主可控的多级分销分润与提现体系，促进市场推广。

1. **代理商招募前台**
   - **内容**：在官网上线“代理商招募/伙伴计划”介绍页面，展示多级分销的收益分配政策。
2. **代理商控制台 (Partner Dashboard)**
   - **内容**：提供独立的登录与管理页面，允许伙伴查看专属推广码/链接、名下邀请用户、分润账单明细，并可在线发起提现申请。
3. **管理端后台集成**
   - **内容**：在 `/dashboard-panel` 增加代理商与财务管理模块，支持管理员审核伙伴身份、配置不同层级的分润比例、审批提现并记录流水。
4. **SQLite 数据库设计适配**
   - **内容**：在 SQLite 数据库中设计并预留 `partners`（伙伴账户表）、`partner_earnings`（分润流水表）和 `payout_requests`（提现申请表）的表结构，为未来的分润功能预留底层数据结构支撑。
